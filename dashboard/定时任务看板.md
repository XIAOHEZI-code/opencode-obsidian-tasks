## 🔧 管理命令

在终端中可用的命令：

### 查看任务状态
```bash
bunx opencode-tasks --status
```

### 列出所有已注册任务
```bash
bunx opencode-tasks --list
```

### 手动运行某个任务
```bash
bunx opencode-tasks --run daily-summary
bunx opencode-tasks --run weekly-report
```

### 查看任务日志
```bash
bunx opencode-tasks --logs daily-summary
```

### 重新加载任务配置
```bash
bunx opencode-tasks --reload
```

### 停止 / 启动调度器
```bash
systemctl --user stop opencode-tasks.timer
systemctl --user start opencode-tasks.timer
```

## ⏱️ Cron 表达式速查

| 表达式 | 含义 |
|--------|------|
| `0 23 * * *` | 每天 23:00 |
| `0 9 * * 0` | 每周日 09:00 |
| `0 9 * * 1-5` | 工作日 09:00 |
| `0 */6 * * *` | 每 6 小时 |
| `30 8 1 * *` | 每月 1 号 08:30 |
| `0 9,18 * * *` | 每天 09:00 和 18:00 |

## 📂 任务文件位置

所有任务文件存放在：

```
~/.config/opencode/tasks/
├── daily-summary.md       # 每日回顾
├── weekly-report.md       # 每周报告
└── your-custom-task.md    # 自定义任务（自行添加）
```

---

## ➕ 添加新任务

1. 在 `~/.config/opencode/tasks/` 下创建新的 `.md` 文件
2. 参考 `daily-summary.md` 的 frontmatter 格式写配置
3. 运行 `bunx opencode-tasks --reload` 加载新任务
4. 用 `bunx opencode-tasks --status` 确认已注册

### 任务模板

```yaml
---
description: 任务描述
schedule: "0 9 * * *"      # cron 表达式
cwd: ~                      # 工作目录
agent: build                # 使用的 OpenCode 代理
permission:
  obsidian_*: "allow"
  read: "allow"
  glob: "allow"
  grep: "allow"
  bash:
    "*": "deny"
    "git log*": "allow"
    "git diff*": "allow"
    "find *": "allow"
    "date *": "allow"
    "pwd": "allow"
  write:
    "*": "deny"
    "<YOUR_VAULT_PATH>/*": "allow"   # 👈 改为你自己的 vault 路径
---

# 任务标题

任务步骤...
```
