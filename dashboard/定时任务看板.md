## 🔧 管理命令

在终端执行以下命令管理定时任务：

| 命令 | 作用 |
|------|------|
| `bunx opencode-tasks --status` | 查看调度器和任务状态 |
| `bunx opencode-tasks --list` | 列出所有任务及下次运行时间 |
| `systemctl --user status opencode-tasks.timer` | 查看 systemd timer 状态 |
| `systemctl --user list-timers | grep opencode` | 查看下次触发时间 |

## 📂 任务文件位置

任务定义文件位于：
- `~/.config/opencode/tasks/daily-summary.md` — 每日复盘
- `~/.config/opencode/tasks/weekly-report.md` — 每周周报

可直接编辑这些 markdown 文件来修改任务内容（prompt）、执行时间（cron）、或权限。

## 🕐 Cron 表达式速查

| 表达式 | 含义 |
|--------|------|
| `0 23 * * *` | 每天 23:00 |
| `0 9 * * 0` | 每周日 09:00 |
| `0 9 * * 1-5` | 工作日 09:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 0 1 * *` | 每月 1 号 00:00 |

## 🛠️ 如何添加新任务

1. 在 `~/.config/opencode/tasks/` 下创建新的 `.md` 文件
2. YAML frontmatter 中设置 `schedule`（cron 表达式）
3. 正文写任务 prompt
4. 调度器每 60 秒自动检测新文件

模板：
```yaml
---
description: 任务描述
schedule: "0 9 * * *"
cwd: ~
agent: build
permission:
  read: "allow"
  bash:
    "git *": "allow"
---

你的任务 prompt 写在这里
```
