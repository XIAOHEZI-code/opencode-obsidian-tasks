## ✨ What It Does / 它做什么

You write notes and commit code every day — but when was the last time you actually *reviewed* them? Valuable insights, forgotten todos, and cross-connection opportunities slip through the cracks.

你每天都在写笔记、提交代码 — 但你上一次真正*回顾*它们是什么时候？许多有价值的洞见、被遗忘的待办事项和交叉关联的机会就此流失。

This project solves that. Two automated tasks powered by OpenCode AI run on schedule, scan your work, and deliver structured, actionable summaries straight into your Obsidian vault.

这个项目解决了这个问题。两个由 OpenCode AI 驱动的自动化任务按计划运行，扫描你的工作产出，生成结构化的、可执行的分析报告，直接写入你的 Obsidian 知识库。

| Task / 任务 | Schedule / 调度 | What It Does / 功能 |
|-------------|-----------------|---------------------|
| 📝 Daily Review / 每日回顾 | Every day 23:00 / 每天 23:00 | Scans today's git commits & Obsidian notes, generates AI review with actionable suggestions / 扫描当日 Git 提交和 Obsidian 笔记，生成带建议的 AI 回顾 |
| 📊 Weekly Report / 每周报告 | Every Sunday 09:00 / 每周日 09:00 | Aggregates weekly activity into structured markdown report / 聚合一周活动，生成结构化 Markdown 周报 |

## 🚀 Quick Start / 快速开始

### Prerequisites / 前置条件

| Tool / 工具 | Purpose / 用途 | Install / 安装 |
|-------------|----------------|----------------|
| [OpenCode](https://github.com/opencode-ai/opencode) | AI agent runtime / AI 代理运行时 | Follow official guide / 跟随官方指南 |
| [Obsidian](https://obsidian.md) | Your knowledge vault / 你的知识库 | [Download](https://obsidian.md/download) |
| [Bun](https://bun.sh) | JS runtime for CLI / CLI 的 JS 运行时 | `curl -fsSL https://bun.sh/install \| bash` |

> **Obsidian Plugin Required / 需要的 Obsidian 插件**: Periodic Notes (or Daily Notes core plugin) — for daily note integration.

### Step 1 — Install opencode-tasks plugin / 安装插件

Add `"opencode-tasks"` to the `plugin` array in your OpenCode config:

```bash
# Edit ~/.config/opencode/opencode.json
# 编辑 ~/.config/opencode/opencode.json
```

```jsonc
{
  "plugin": [
    // ... other plugins ...
    "opencode-tasks"
  ]
}
```

### Step 2 — Copy task files / 复制任务文件

```bash
git clone https://github.com/XIAOHEZI-code/opencode-obsidian-tasks.git
cp opencode-obsidian-tasks/tasks/*.md ~/.config/opencode/tasks/
```

### Step 3 — Customize paths / 自定义路径

Edit the task files and replace the placeholders:

| Placeholder / 占位符 | Replace with / 替换为 | Example / 示例 |
|----------------------|----------------------|----------------|
| `<YOUR_VAULT_PATH>` | Your Obsidian vault path / 你的 Obsidian 知识库路径 | `~/Documents/MyVault` |
| `~/projects/` (in task body) | Your git projects directory / 你的 Git 项目目录 | `~/Code/` or `~/Desktop/project/` |

```bash
# Quick sed replace (adjust paths as needed)
# 快速替换（根据实际路径调整）
sed -i 's|<YOUR_VAULT_PATH>|~/Documents/MyVault|g' ~/.config/opencode/tasks/*.md
```

### Step 4 — Install the scheduler / 安装调度器

```bash
bunx opencode-tasks --install
```

This creates systemd user-level timer and service units that check for due tasks every 60 seconds.

### Step 5 — Verify / 验证

```bash
bunx opencode-tasks --status
```

You should see both tasks listed with their next scheduled run times.
你应该看到两个任务列出，并显示下次运行时间。

## 📋 Task Breakdown / 任务详解

### 📝 daily-summary.md

| Step / 步骤 | Action / 动作 |
|-------------|---------------|
| **1. Scan Git** | Walks `~/projects/`, finds repos with today's commits, gathers messages, diffs, and stats. / 遍历 `~/projects/`，找出当天有提交的仓库，收集提交信息、差异和统计。 |
| **2. Scan Obsidian** | Finds notes created/modified today, reads the daily note, gathers tags and backlinks. / 查找当天创建/修改的笔记，读取每日笔记，收集标签和反向链接。 |
| **3. AI Review** | Synthesizes findings into structured markdown: git table, key changes, notes table, suggestions, tags. / 将发现综合为结构化 Markdown：Git 表格、关键变更、笔记表格、建议、标签。 |
| **4. Write** | Appends the review to today's daily note via Obsidian API. Falls back to direct `.md` write if Obsidian is closed. / 通过 Obsidian API 将回顾追加到每日笔记。如果 Obsidian 关闭，降级为直接写入 `.md` 文件。 |

### 📊 weekly-report.md

| Step / 步骤 | Action / 动作 |
|-------------|---------------|
| **1. Week Range** | Calculates the 7-day window. / 计算 7 天时间窗口。 |
| **2. Weekly Git** | Aggregates all git activity across repos for the week. / 聚合一周内所有仓库的 Git 活动。 |
| **3. Weekly Notes** | Finds all notes modified within the week, identifies trends and top tags. / 查找一周内修改的所有笔记，识别趋势和热门标签。 |
| **4. Generate** | Builds a comprehensive markdown report with stats, highlights, insights, and cross-references. / 生成包含统计、亮点、洞察和交叉引用的综合 Markdown 报告。 |
| **5. Save** | Writes to `<YOUR_VAULT_PATH>/Reports/Weekly/YYYY-WW.md`. / 写入 `<YOUR_VAULT_PATH>/Reports/Weekly/YYYY-WW.md`。 |

## 🛠️ Customization / 自定义

### Change schedule / 修改调度时间

Edit the `schedule` field in the task's frontmatter. Uses standard cron syntax.

```yaml
schedule: "0 20 * * *"    # 8:00 PM instead of 11:00 PM
schedule: "0 9 * * 1-5"   # Weekdays only
schedule: "0 */6 * * *"   # Every 6 hours
```

> 💡 Use [crontab.guru](https://crontab.guru/) to test cron expressions.

### Change AI model / 切换 AI 模型

The model is configured globally in `~/.config/opencode/opencode.json`. The task files intentionally omit the model field so your global setting applies. To override per-task:

```yaml
model: "your-preferred-model"
```

### Add custom tasks / 添加自定义任务

1. Create a new `.md` file in `~/.config/opencode/tasks/`
2. Copy the frontmatter template from a working task
3. Write your task steps in the body
4. Run `bunx opencode-tasks --reload`

### Customize output format / 自定义输出格式

Edit Step 3 (daily-summary) or Step 4 (weekly-report) in the task body. The markdown template is plain text — change headings, add sections, reorder as you like.

## 📂 File Structure / 文件结构

```
~/.config/opencode/
├── opencode.json                  # ← add "opencode-tasks" to plugin array
└── tasks/
    ├── daily-summary.md           # Daily review task definition
    ├── weekly-report.md           # Weekly report task definition
    └── your-custom-task.md        # Add your own! / 自行添加！

~/.config/systemd/user/
├── opencode-tasks.service         # Auto-created by --install
└── opencode-tasks.timer           # Checks every 60s for due tasks

<YOUR_VAULT_PATH>/                 # Your Obsidian vault / 你的 Obsidian 知识库
├── Daily/                         # Daily notes (Periodic Notes plugin)
│   └── YYYY-MM-DD.md              # ← AI review appended here
└── Reports/
    └── Weekly/
        └── YYYY-WW.md             # ← Weekly report saved here
```
