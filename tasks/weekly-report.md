# 📊 Weekly Report

Aggregate the entire week's activity into a comprehensive markdown report saved directly to your Obsidian vault.

## Step 1 — Determine the Week Range

Calculate the date range for the past 7 days (Monday through Sunday, or your preferred week definition).

```bash
# Get today's date and 7 days ago
date +%Y-%m-%d
date -d '7 days ago' +%Y-%m-%d
```

Use these dates to scope all subsequent scans.

## Step 2 — Collect Weekly Git Activity

Scan `~/projects/` (customize this path) for all commits made in the past 7 days.

```bash
# Scan all repos for weekly activity
find ~/projects/ -name .git -type d -maxdepth 3 | while read gitdir; do
  repo=$(dirname "$gitdir")
  echo "=== $repo ==="
  git -C "$repo" log --oneline --since="7 days ago" --all 2>/dev/null
  echo ""
done
```

Gather per-repo:
- Total commits this week
- Files changed (aggregated)
- Lines added / lines removed
- Most active branch

## Step 3 — Collect Weekly Obsidian Activity

Search the vault for notes created or modified in the past 7 days.

- Find all notes modified within the week range
- List new notes created vs. existing notes updated
- Identify the most-used tags this week
- Find notes with the most backlinks added

## Step 4 — Generate Weekly Report

Synthesize everything into a structured report:

```markdown

## 🖥️ Git Activity

| Repo | Commits | Files | +Lines | -Lines | Top Branch |
|------|---------|-------|--------|--------|------------|
| ...  | ...     | ...   | ...    | ...    | ...        |

### 🏆 Most Active Repo
**repo-name** — N commits across M files

### 📝 Commit Highlights
- `abc1234` Brief description of important commit
- `def5678` Brief description of important commit

---

## 📓 Obsidian Activity

| Metric | Count |
|--------|-------|
| New Notes | N |
| Modified Notes | M |
| Total Tags Used | T |

### 🆕 New Notes This Week
| Note | Tags |
|------|------|
| ...  | ...  |

### ✏️ Most Edited Notes
| Note | Edits |
|------|-------|
| ...  | ...   |

### 🏷️ Top Tags This Week
`#tag1` (N uses) · `#tag2` (M uses) · `#tag3` (T uses)

## 💡 Weekly Insights

- 🔥 **Trending Topic**: ...
- 🔗 **Link Suggestions**: Notes that could benefit from cross-linking
- 🧹 **Vault Hygiene**: Orphaned notes, broken links to fix
- 📌 **Next Week Focus**: Based on this week's patterns

---

## 🔗 Cross-References

- Daily summaries: [[YYYY-MM-DD]], [[YYYY-MM-DD]], ...
- Related projects: [[Project A]], [[Project B]]

---

*Generated by OpenCode weekly-report task · YYYY-MM-DD HH:MM*
```

## Step 5 — Save the Report

Write the report to your vault:

```
<YOUR_VAULT_PATH>/Reports/Weekly/YYYY-WW.md
```

- Use `write` to create the file (allowed for `<YOUR_VAULT_PATH>/*`)
- If the Reports/Weekly directory doesn't exist, create it first
- Link back to individual daily notes when possible

### ⚠️ Fallback: When Obsidian API Is Unavailable

1. Use `find` + `ls` to scan markdown files modified in the past 7 days:
   ```bash
   find <YOUR_VAULT_PATH> -name "*.md" -mtime -7 -type f
   ```
2. Parse filenames and modification times manually
3. Write the report directly to the `.md` file
4. Include a note: `*(generated in offline mode)*`

## Reference

- **Schedule**: `0 9 * * 0` — runs every Sunday at 9:00 AM
- **Agent**: `build` — uses the build agent for filesystem + Obsidian access
- **Vault Path**: Replace `<YOUR_VAULT_PATH>` with your actual Obsidian vault path
- **Git Path**: Adjust `~/projects/` in Step 2 to match where your repositories live
- **Report Location**: Reports are saved to `<YOUR_VAULT_PATH>/Reports/Weekly/` by default

## Notes

- The weekly report automatically cross-references daily summaries when available
- Works in offline mode by scanning `.md` files directly
- Customize the report template in Step 4 to fit your preferred format
- You can change the schedule to any cron expression (e.g., `0 18 * * 5` for Friday evenings)
