# 📝 Daily Summary

Scan today's activity and generate an AI-powered daily review that gets written into your Obsidian daily note.

## Step 1 — Scan Today's Git Activity

Scan `~/projects/` (customize this path to wherever your git repos live) for commits made today.

```bash
# Get today's date in ISO format
date +%Y-%m-%d

# Scan git activity across all repos in ~/projects/
find ~/projects/ -name .git -type d -maxdepth 3 | while read gitdir; do
  repo=$(dirname "$gitdir")
  echo "=== $repo ==="
  git -C "$repo" log --oneline --since="midnight" --all 2>/dev/null
  echo ""
done
```

Gather:
- Which repos had commits today
- Commit messages and changed files
- Lines added/removed per repo (use `git diff --stat`)

## Step 2 — Scan Today's Obsidian Notes

Use Obsidian API to find notes created or modified today.

- Search for today's date (`YYYY-MM-DD`) in filenames and content
- Read the daily note if it already exists
- Gather tags used today, new files created, backlinks modified

## Step 3 — Generate AI-Powered Daily Review

Synthesize findings into a structured review with the following format:

```markdown
## 🤖 AI Daily Review — YYYY-MM-DD

### 📊 Git Activity
| Repo | Commits | Files Changed | Summary |
|------|---------|---------------|---------|
| ...  | ...     | ...           | ...     |

### 🔑 Key Changes
- **repo-name**: Brief description of the most important change
- ...

### 📝 Notes Created / Modified
| Note | Action | Tags |
|------|--------|------|
| ...  | ...    | ...  |

### 💡 Suggestions
- 🧹 **Housekeeping**: ...
- 🔗 **Connect**: ...
- 📌 **Follow-up**: ...

### 🏷️ Today's Tags
`#tag1` `#tag2` ...

---
*Generated by OpenCode daily-summary task*
```

## Step 4 — Write to Daily Note

Write the generated review into today's Obsidian daily note.

- Use `obsidian_get_daily_note` to check if the daily note exists
- Use `obsidian_patch_note` or `obsidian_append_to_daily_note` to append the review
- If the daily note already has an AI review section, update it instead of duplicating

### ⚠️ Fallback: When Obsidian API Is Unavailable

If Obsidian is not running or the Obsidian API calls fail, fall back to direct file writing:

1. Determine the daily note path based on your vault's naming convention
   - Periodic Notes plugin: `<YOUR_VAULT_PATH>/Daily/YYYY-MM-DD.md`
   - Daily Notes core plugin: `<YOUR_VAULT_PATH>/YYYY-MM-DD.md`
2. Use `read` to check if the file exists
3. Use `write` (allowed for `<YOUR_VAULT_PATH>/*`) to create or append to the file
4. Include a small note: `*(offline mode — written directly to file)*`

## Reference

- **Schedule**: `0 23 * * *` — runs every day at 11:00 PM
- **Agent**: `build` — uses the build agent for filesystem + Obsidian access
- **Vault Path**: Replace `<YOUR_VAULT_PATH>` with the actual path to your Obsidian vault (e.g., `~/Documents/MyVault`)
- **Git Path**: Adjust `~/projects/` in Step 1 to match where your repositories live

## Notes

- The task runs even if Obsidian is closed, thanks to the file-writing fallback
- All data stays local — nothing is sent to external services
- Customize the output format in Step 3 to match your preferred daily review structure
